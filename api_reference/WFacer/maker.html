Automatic cluster expansion workflow maker.

name#

The name of the cluster expansion project.

Note

Since the underscore (“_”) will be used to separate the fields in job names, it should not appear in the project name here!

Type:

str

options#

A dictionary including all options to set up the automatic workflow. For available options, see docs in preprocessing.py.

Type:

dict

make(prim, last_document=None, add_num_iterations=None)[source]#

Make the workflow.

Parameters:

  • prim (Structure) – A primitive cell structure (no need to be reduced) with partial occupancy on some sub-lattice. This defines the lattice model of your cluster expansion.

  • last_document (CeOutputsDocument) – optional A cluster expansion outputs maker to continue from. Used for warm restart. Will run at maximum another “max_iter” number of iterations. Default is None, which means to start from scratch.

  • add_num_iterations (int) – optional When the last document has reached the maximum allowed number of iterations, add this many more iterations. Default is None. When given None or 0, will simply double max_iter in options.

Returns:

The iterative automatic cluster expansion workflow.

Return type:

Flow
